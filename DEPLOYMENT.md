# 🚀 Deployment Guide - SSL Expiry Monitor Dashboard

This guide provides simple, step-by-step instructions to deploy the SSL Expiry Monitor on port `8800` and keep it running continuously on your server.

---

## Option 1: Docker Compose (Recommended - Quickest & Cleanest)

If your server has Docker and Docker Compose installed, this is the easiest deployment option.

### Steps:
1. **Clone/Copy Project Files** to your VPS.
2. **Configure Environment Variables**:
   Edit the `.env` file to set your SMTP details and admin password:
   ```bash
   nano .env
   ```
3. **Start the Container** in detached mode:
   ```bash
   docker-compose up -d --build
   ```
4. **Access the Dashboard**:
   Open `http://<your-server-ip>:8800` in your web browser.

---

## Option 2: PM2 Process Manager (Cross-Platform)

PM2 is a production process manager that can manage and keep Python applications alive forever. It works on both Linux and Windows.

### Steps:
1. **Install Node.js & PM2**:
   - **Linux**:
     ```bash
     curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
     sudo apt-get install -y nodejs
     sudo npm install -g pm2
     ```
   - **Windows**: Download and run the Node.js installer from nodejs.org, then run:
     ```powershell
     npm install -g pm2
     ```

2. **Install Python Dependencies**:
   ```bash
   pip install -r requirements.txt
   ```

3. **Start the Process**:
   ```bash
   pm2 start ssl_monitor.py --name "ssl-monitor" --interpreter python -- --web
   ```
   *(Since we updated the default port in the script, it automatically listens on port `8800`)*.

4. **Set Up Startup Persistence**:
   - **Linux**:
     ```bash
     pm2 startup
     # Run the command generated by the output of the line above, then run:
     pm2 save
     ```
   - **Windows**: Use `pm2-windows-startup` or `pm2-windows-service` to run PM2 as a background service.

## Option 3: Systemd Service (Standard Linux VPS)

To deploy as a native systemd service on Ubuntu/Debian/CentOS VPS.

### Steps:
1. **Create the Service File**:
   ```bash
   sudo nano /etc/systemd/system/ssl-monitor.service
   ```
2. **Paste the Configuration**:
   *(Replace `/path/to/ssl-expiry-checker` with your absolute folder path, and verify the path to `python`)*
   ```ini
   [Unit]
   Description=SSL Expiry Monitor Web Dashboard
   After=network.target

   [Service]
   Type=simple
   User=root
   WorkingDirectory=/path/to/ssl-expiry-checker
   ExecStart=/usr/bin/python3 ssl_monitor.py --web
   Restart=always
   RestartSec=5
   Environment=PYTHONUNBUFFERED=1

   [Install]
   WantedBy=multi-user.target
   ```
3. **Start and Enable Service**:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable ssl-monitor
   sudo systemctl start ssl-monitor
   ```
4. **Check Service Status**:
   ```bash
   sudo systemctl status ssl-monitor
   ```

## Option 4: Windows Service Deployment (Using NSSM)

If you are deploying on a Windows Server VPS.

### Steps:
1. Download **NSSM** (Non-Sucking Service Manager) from [nssm.cc](https://nssm.cc/).
2. Open PowerShell or Command Prompt as Administrator and run:
   ```cmd
   nssm install SSLMonitor
   ```
3. In the GUI popup window:
   - **Path**: Path to python executable (e.g., `C:\Python310\python.exe`)
   - **Startup directory**: Path to your application directory (e.g., `E:\Tagid\ssl-expiry-checker`)
   - **Arguments**: `ssl_monitor.py --web`
4. Click **Install service**.
5. Start the service:
   ```cmd
   nssm start SSLMonitor
   ```
